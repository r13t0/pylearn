# Validation → Classification Pipeline

## Purpose

Ensure input is valid before assigning it to a category.

## Flow

Input
 ↓
Validation
 ↓
Classification
 ↓
Output

## Recognition Clues

- "If invalid..."
- "Assign a grade."
- "Choose a category."

## Reusable Algorithm

Receive input

Check validity

If invalid:
    Report error

Else:
    Determine category

Display result

## Sample Code

```python
score = float(input("Score: "))

if score < 0 or score > 100:
    print("Invalid score")
elif score >= 90:
    print("A")
elif score >= 80:
    print("B")
else:
    print("C")


One important observation

You've now learned three fundamental decision patterns:

Pattern	Question it answers
Validation	"Is this input acceptable?"
Selection	"Which action should I take?"
Classification	"Which category does this belong to?"

These three patterns will appear repeatedly throughout your programming journey. Recognizing which question a problem is asking is often the fastest way to identify the underlying pattern.
```
